Endpoint Detection and Response (EDR)

About Planqx EDR

This is an open source EDR basically advance Anti-virus tool, build with the Cloud-Native Architecture, let Explore how this works - start with client side, the lightweight forwarder named Planqx Sensor, This sensor serves as the backbone of the EDR system which forward the Data, system logs, and some internal Telemetry data to the server, this entirely monitor the system with both user and kernel mode, network level motive to safe guard all side of threat's, and this sensor multiple technique like API hooking, ELAM (Early launch anti-malware), ETW, kernel level monitoring custom Kernel driver, which allows to monitor low-level system activities that might bypass traditional user-mode defenses like other AV, this system integrates advanced logging and alerting mechanisms empowering incident response teams with detailed log analysis capabilities for effective threat detection and resolution.

Special Features and Outcomes

This EDR system is designed to protect enterprise systems from threats by detecting malicious activity at the endpoint base level.

Real-Time Threat Detection – Detects malicious activities such as process injections, malware execution, unauthorized access.

Network Level Detection – Analyse the whole network Traffic like an firewall, prevent from Malicious - Traffic, Domains, IP's, DDos attack, System scanning (like port scaning), etc...

Scability – Easily handles large-scale deployments without compromising performance control over 5000 endpoints concurrently as of now.

Exploit suggestor – Analyse the Endpoints system logs to identify the potential vulnerbilities by comparing its patch level against database of known vulnerbilities using CVE's, Exploit Db and IOC's and update the Endpoints using RPC/WMI.

Windows Powershell authenticator

About

The Windows Authenticator is a security script designed to provide an additional layer of authentication for Windows PowerShell and Command Prompt users. It ensures advanced-level security by authenticating users before granting access to sensitive operations or resources.

Why Windows Authenticator?

While Windows is generally considered secure, it may lack of the authentication features commonly found in Linux systems, such as the sudo or Root user The Windows Authenticator addresses this gap by introducing an authentication mechanism similar to the one found in Linux systems, ensuring enhanced security and user control.

let's discuss How It Works ?

Registry Configuration – The script sets a registry key to control access to PowerShell and Command Prompt.

User Authentication – Users are prompted to enter their credentials when attempting to access PowerShell or Command Prompt.

Credential Validation – The entered credentials are validated against predefined username and password variables.

Access Granting – If the credentials match, access to PowerShell and Command Prompt is granted. Otherwise, access is denied.
