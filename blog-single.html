What is TURN?

TURN (Traversal Using Relays around NAT) is a protocol that assists in traversing Network Address Translators (NATs) or firewalls to facilitate peer-to-peer communications. It is an extension of the STUN (Session Traversal Utilities for NAT) protocol and is defined in RFC 8656.

How do I use TURN?

To use the TURN protocol in your application, you need a tool or library. Most users of TURN use it as part of a WebRTC library, such as the one in their browser or as part of Pion, webrtc-rs, or libwebrtc. Some of these libraries offer TURN client support directly, like Pion for Golang and webrtc-rs for Rust.

Key Concepts of TURN

  • NAT (Network Address Translation): A method routers use to map multiple private IP addresses to a single public IP.
  • TURN Server: A relay server that acts as an intermediary for traffic between clients behind NATs.
  • TURN Client: An application that uses the TURN protocol to communicate through a TURN server.
  • Allocation: When a TURN server reserves a unique IP and port for a client.
  • Relayed Transport Address: The IP address and port on the TURN server that peers use to send data to the client.

How TURN Works

  1. A TURN client sends an Allocate request to a TURN server.
  2. The server creates an allocation and returns a relayed transport address to the client.
  3. The client shares this relayed address with its peers.
  4. The TURN server forwards incoming data to the client and sends outgoing data from the client to the peer.

TURN vs VPN

TURN is often compared to VPNs, but they serve different purposes:

  • VPN: A general-purpose tool that encrypts all internet traffic, routing it through a server to enhance privacy and security.
  • TURN Server: A specialized tool used mainly for real-time communication, acting as a relay only for specific application traffic.

Why is TURN Useful?

TURN is valuable in scenarios where direct peer-to-peer communication is blocked due to NAT or firewall restrictions. It provides:

  • NAT Traversal: Enabling connections between peers behind NATs.
  • Firewall Bypassing: Allowing communication in environments with strict firewall policies.
  • Consistent Connectivity: Offering a fallback method when direct connections fail.
  • Privacy: Hiding actual IP addresses by relaying traffic through a TURN server.
  • Applications: Essential for VoIP, video conferencing, online gaming, and IoT communication.
